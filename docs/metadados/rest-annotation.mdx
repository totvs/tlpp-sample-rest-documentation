# Metadados da annotation REST

As annotations `@Get` e `@Put` aceitam metadados que o motor REST-DOC converte em OpenAPI. Alguns são **funcionais** (registram a rota); outros são **informativos** (só documentação).

## Propriedades

| Metadado | Para que serve |
|----------|----------------|
| **`endpoint`** | Path da rota REST — obrigatório para registrar o serviço |
| **`description`** | Origem da documentação — ver [description](/docs/tlpp/rest/metadados/description) |
| **`title`** | Título legível do endpoint no OpenAPI |
| **`params`** | Parâmetros (`query`, `path`, `header`) como string JSON |
| **`responses`** | Respostas HTTP (`statusCode`, `description`) como string JSON |

## `params` e `responses` inline

Quando a documentação é simples, você pode declarar parâmetros e respostas **direto na annotation**, sem função DOC:

```tlpp
@Get(;
  endpoint="/rest/sample/doc/basic/02",;
  description="...",;
  title="DOC simples 02",;
  params='[{"name":"param1","in":"query",...}]',;
  responses='[{"statusCode":200,"description":"OK"},...]';
)
```

Em documentações mais ricas (função `_DOC`), o JSON suporta campos extras como `minimum`, `maximum`, `values` e `requestBody`.
