# Visão geral dos metadados

O **REST-DOC** do tlppCore lê **metadados** no seu código TLPP e monta a especificação **OpenAPI** (arquivo YAML/JSON que descreve endpoints, parâmetros e respostas). Este repositório demonstra cada tipo de metadado que o motor entende.

:::info Pré-requisitos (trilha REST)
Antes de documentar, você precisa de **pelo menos um endpoint funcionando**:

1. [Quickstart REST](/docs/tlpp/rest/primeiros-passos/quickstart)
2. [Annotations](/docs/tlpp/rest/configuracoes/annotations) — ou [Sem annotation](/docs/tlpp/rest/configuracoes/sem-annotation) para rotas dinâmicas

Metadados **não substituem** a rota — eles só alimentam a documentação OpenAPI.
:::

## Trilha recomendada (menu Doc Generate)

1. **description** + exemplo 01 — caso mais simples  
2. **Annotation REST** + exemplo 02 — `title`, `params`, `responses` inline  
3. **JSON DOC** + exemplos 04 e 05 — função dedicada `_DOC`  
4. **i18n** + exemplo 03 — textos multilíngue  
5. **Componentes** → exemplo completo → **Avisos e dicas**  
6. **Geração OpenAPI** — `tlpp.doc.generate()`, snapshot YAML, mapa do repositório  

Siga o menu lateral na ordem — cada conceito é seguido do exemplo correspondente.

## O que existe

| Metadado | Para que serve |
|----------|----------------|
| **`endpoint`** | Registra a rota REST no AppServer (comportamento funcional) |
| **`description`** | Aponta a documentação do endpoint — texto, função DOC ou ID i18n |
| **`title`**, **`params`**, **`responses`** | Campos OpenAPI declarados direto na annotation (`@Get`, `@Post`…) |
| **JSON da função `_DOC`** | Documentação rica retornada por uma função TLPP |
| **`description1`…`N`** | Descrição longa dividida em blocos numerados no JSON |
| **`translate:…`** | Textos traduzíveis dentro do JSON DOC |
| **`TLPP COMPONENT`** | Schemas reutilizáveis referenciados no JSON |
| **`tlpp.doc.List()`** | Associa documentação a rotas criadas dinamicamente |
| **`tlpp.doc.generate()`** | Exporta tudo para YAML/JSON OpenAPI |

## Quatro formas de documentar

1. **Texto na `description`** — uma linha na annotation
2. **Metadados inline na annotation** — `title`, `params`, `responses`
3. **Função dedicada `_DOC`** — JSON em função separada (recomendado para APIs reais)
4. **Rotas dinâmicas** — `tlpp.doc.List()` liga endpoint + método + função DOC

## Regras importantes

1. **Prefira `JsonObject():New()`** em funções DOC — evita JSON inválido
2. **Multi-linha:** não pule números em `descriptionN`
3. **Geração:** o 1º parâmetro de `tlpp.doc.generate` deve ser `'swagger'` (nome legado do motor)
